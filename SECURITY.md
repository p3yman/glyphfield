# Security Policy

## Supported versions

Security fixes are provided for the latest published version.

## Reporting a vulnerability

Use GitHub private vulnerability reporting for `p3yman/glyphfield`. Do not open a public issue containing exploit details, credentials, or private data. Include affected versions, impact, reproduction steps, and any proposed mitigation. Expect an acknowledgement within seven days.

Do not send npm tokens or other credentials. This project uses npm trusted publishing and does not require a long-lived publishing token.
